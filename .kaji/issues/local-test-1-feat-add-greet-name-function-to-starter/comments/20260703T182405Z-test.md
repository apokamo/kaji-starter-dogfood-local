### 設計の要点

1. **What**: `starter_app` に `greet(name: str) -> str` を追加。`"Hello, {name}!"` を返す純粋関数。
2. **Why**: `hello()` は挨拶対象を指定できない。名前引数を取る関数で TDD の 1 周（正常系 + 異常系）を最小コードで体験できるようにする。既存 `hello()` は不変にするため新規関数を追加する方針。
3. **Constraints**: 実装先は既存 `src/starter_app/__init__.py`。mypy strict / 型注釈必須 / 外部依存追加なし。空文字・空白のみは `ValueError`。

### テスト戦略

- **変更タイプ**: 実行時コード変更（ドメインロジック + バリデーション分岐）
- **Small**: 正常系（`"Alice"` → `"Hello, Alice!"`）/ トリム（`"  Bob  "` → `"Hello, Bob!"`）/ 異常系（空文字 → `ValueError`）/ 異常系（空白のみ → `ValueError`）
- **Medium / Large**: 純粋関数で外部依存なし・CLI はスコープ外のため、testing-convention 4 条件に沿って恒久テストを追加しない旨を設計書に明記
